THE PROBLEM THIS SOLVES:
IndiGround Services manages 8 gates at a regional hub. Ground ops teams 
(fuel, baggage, catering) work in silos causing Turnaround Time (TAT) bloat. 
Every minute of delay = ‚Çπ5,400 in penalties. TarmacIQ predicts TAT, detects 
bottlenecks, and quantifies financial risk before the flight even lands.

BACKEND (Python Flask):

Create these API endpoints:

POST /predict
  - Input: flight features
  - Run linear regression model (train on synthetic data if no CSV provided)
  - Calculate Baggage_Duration, Fuel_Duration, Catering_Duration
  - Identify bottleneck = max(Baggage_Share, Fuel_Share, Catering_Share)
  - Return: predicted_TAT, bottleneck, penalty_risk

POST /montecarlo
  - Input: flight features + predicted sub-process durations
  - Run 1000 simulations with numpy random normal noise
  - Return: P50, P75, P90 TAT values and penalty amounts, 
    bottleneck_consistency percentage, histogram data array

GET /gates
  - Return current state of all 8 gates (free/active/clearing, 
    flight assigned, TAT progress)

GET /analytics
  - Return historical summary stats for Page 3 charts
